# ADR-0020: 采用边缘桥接方式完整集成 Home Assistant

状态：于 2026-07-17 接受，按阶段实施。

## 背景

Home Assistant 已经覆盖大量品牌、设备类型、本地协议和厂商云服务。复用它的设备
接入能力，可以让 AetherEdge 把研发重点放在智能体意图、安全策略、确定性执行和
可验证结果上，而不是重复维护每个厂商的私有接入代码。

这项集成不能被降级为一个普通数值通道。Home Assistant 暴露区域、设备、实体、
服务、可用性和异构属性；其中包含布尔值、枚举、文本、时间以及结构化数据。现有
AetherEdge 点位与共享内存主要承载有限数值，无法无损表达完整模型。

这项集成也不能成为任意服务代理。把智能体输入直接转发给 Home Assistant 的
`call_service`，会绕过 AetherEdge 的能力声明、权限、风险分级、确认、幂等、
前置条件和审计边界。

因此，“完整集成”是指完整同步 Home Assistant 可公开提供的设备目录和状态，并
通过可治理的能力目录逐步扩大可写覆盖面；它不表示无条件开放所有服务，也不表示
Home Assistant 的一次服务调用成功已经证明物理动作完成。

## 决策

Home Assistant 以可选边缘桥接扩展接入 AetherEdge：

```text
Home Assistant
      |
      v
extensions/home-assistant-bridge
      |
      v
厂商中立端口与应用用例
      |
      +--> 外部设备目录与异构状态
      |
      +--> 显式映射的数值投影 --> SHM
      |
      +--> 经治理的命令与执行证据
```

桥接实现位于 `extensions/home-assistant-bridge`，由 `services/io` 这一组合根按
配置选择和装配。核心包不得依赖该扩展、Home Assistant 数据传输对象、客户端库
或服务名称。

默认发行仍可在没有 Home Assistant、云端、消息代理和外部数据库的单台 Linux
主机上运行。桥接能力默认关闭；默认验证路径使用固定样例和模拟服务，不要求真实
Home Assistant 实例。

## 权威边界

权威按数据含义划分，而不是按传输位置划分：

| 数据或决策 | 权威 |
| --- | --- |
| 经 Home Assistant 委托接入实体的目录、可用性和可观察实际状态 | Home Assistant |
| 接入配置、实体映射、允许的能力、安全策略、授权、确认和审计 | AetherEdge |
| 已接受并映射到 AetherEdge 点位空间的有限数值实时投影 | SHM |
| 云端设备视图、检索索引和历史分析 | 派生副本，不是控制或实时状态权威 |

SHM 不保存 Home Assistant 的完整状态，也不得把文本、枚举或结构化属性伪装成
数值。只有经过显式配置、单位换算和有限值校验的数值才能进入点位投影。对外查询
SHM 时，它是该数值投影的权威；不得据此声称它是原始实体的完整状态。

Home Assistant 的状态是该委托接入路径的软件事实。服务调用被接受只证明
Home Assistant 接受了请求，不证明设备已经执行。物理确认必须来自满足预期后置
条件的后续状态观测，或来自设备与集成明确提供的更强证据；缺少证据时结果必须为
“确认未知”，不能推断成功。

原生 AetherEdge 设备、规则、安全联锁和采集通路不改变既有权威。Home Assistant
或桥接扩展失联不得停止这些本地行为。

## 厂商中立核心契约

核心只新增表达外部设备提供者所必需的厂商中立契约，不新增 Home Assistant
专用类型：

1. **外部网关身份**：稳定网关标识、连接代次、观测时间、健康状态和来源证明。
2. **设备拓扑**：区域、设备、实体及其关系；稳定外部标识与显示名称分离。完整
   快照和增量都带代次、修订、摘要和删除墓碑。
3. **异构值**：显式区分未知、不可用、布尔值、有符号整数、无符号整数、十进制、
   文本、枚举、时间、时长、字节及受深度和大小限制的集合。数值同时携带单位、
   范围和精度声明。不得以无边界的任意对象代替契约。
4. **能力目录**：每项查询或命令声明目标类型、参数与结果模式、风险、权限、
   确认、幂等、前置条件、有效期、速率限制、审计要求、可用性和来源修订。
5. **同步协议**：完整快照、增量、游标、确认、重置、删除墓碑和代次隔离。旧连接
   代次的事件不能推进新连接。
6. **命令协议**：命令标识、稳定目标、能力标识与修订、异构参数、截止时间、
   幂等键、前置条件、调用者和策略证据。
7. **执行证据**：明确区分已请求、策略已授权、已派发、Home Assistant 已接受、
   目标状态已观测、物理已确认、确认未知、已拒绝、已过期和执行失败。
8. **失败分类**：区分暂时故障、永久故障、不支持、策略拒绝、过期和状态冲突，
   由应用层决定是否允许重试。

Home Assistant 的实体领域、服务名称和原始属性只存在于扩展内部。扩展把它们
映射到上述契约，并通过共享的一致性测试证明身份稳定性、值转换、能力声明和错误
分类。核心不得按固定实体领域枚举分支。

所有实体都可以通过受限、厂商中立的只读表示被发现。写能力只有在内置或已签名的
映射配置声明参数、风险和预期后置条件后才会出现。未知实体或未知服务保持只读并
默认拒绝写入，这不妨碍完整发现，也不会以“兼容”为由放弃安全边界。

## 连接、同步与背压

扩展只使用 Home Assistant 公布的接口，不读取其数据库或内部文件。初次连接和
断线恢复遵循以下顺序：

1. 使用密钥引用解析凭据并完成认证；
2. 建立事件订阅并为当前连接分配新代次；
3. 读取区域、设备、实体、状态和服务的完整快照；
4. 合并快照期间缓存的较新事件；
5. 原子发布带摘要的完整代次；
6. 继续发布有序增量，并用心跳和超时检测半开连接。

重连使用有上限的指数退避和随机扰动。认证拒绝、权限不足和不兼容接口属于永久
故障，不得无限快速重试。重连后必须重新同步完整快照，不能把本地缓存当作
Home Assistant 当前事实。

队列全部有界，并按信息类别隔离：

- 高频状态更新可以按稳定实体标识合并，只保留同一连接代次内的最新值；
- 拓扑增量溢出时废弃该增量游标并请求完整重同步，不能静默丢失删除或重命名；
- 命令、策略决定和执行证据不得被状态合并覆盖，也不得在未记录失败时丢弃；
- 过载必须产生可观测指标和审计事件，但原始敏感属性不能进入日志。

断线期间不接受依赖最新 Home Assistant 状态且无法验证前置条件的新命令。只有
明确声明幂等且仍在有效期内的命令才可能按策略重试；非幂等动作不得自动重放。

## 密钥与隐私

配置保存 Home Assistant 地址、传输安全设置和密钥引用，不保存明文令牌。令牌、
刷新凭据、客户端私钥和完整认证响应不得进入：

- AetherCloud；
- 日志、指标标签和追踪；
- 审计载荷；
- 智能体提示词、工具结果或错误详情；
- 拓扑快照、能力目录和共享内存。

密钥由本地密钥提供者按最小权限解析，并支持轮换和撤销。审计只记录密钥引用、
凭据代次和脱敏连接结果。生产连接默认要求受验证的传输加密；跳过证书验证只能是
明确标注的开发选项，不能成为默认行为。

属性采用允许清单、大小上限和敏感性分类。摄像内容、位置、人员存在、媒体内容及
厂商自定义高基数字段不会因为 Home Assistant 可以返回它们就自动进入云端或
智能体上下文。

## 命令安全

所有写操作通过统一应用命令接口进入，智能体、命令行和网络接口没有旁路。命令
执行前必须验证：

- 目标和能力仍存在，能力修订与请求一致；
- 参数满足类型、范围、枚举和单位约束；
- 调用者拥有所需权限；
- 风险和确认策略已经满足；
- 截止时间、幂等键和前置状态有效；
- 当前连接代次可以安全执行该动作。

扩展只调用能力映射明确允许的目标和服务。系统不提供任意 `call_service` 代理，
不接受由智能体拼接领域、服务和原始数据后直接转发。任何未声明组合都默认拒绝。

Home Assistant 接受请求、观察到预期状态以及取得物理确认是三个独立阶段。每次
阶段变化都携带命令标识、连接代次、观测时间和脱敏证据摘要。延迟到达的状态不能
把已过期或已被新命令取代的命令改写为成功。

## 云端边界

AetherCloud 不直接连接 Home Assistant，也不持有其地址、令牌或刷新凭据。
AetherEdge 可以在未来通过 AetherContracts 定义的厂商中立协议向云端报告：

- 脱敏后的外部网关观测；
- 设备拓扑快照和增量；
- 能力目录；
- 允许上送的状态投影；
- 经治理命令的阶段性执行证据。

云端可以检索派生目录或创建经治理的期望任务，AetherEdge 仍拥有最终的本地策略
决定权。CloudLink 在相应公共契约和一致性测试发布前不得私自承载
Home Assistant 数据传输对象或任意服务调用。

云端失联不改变 Home Assistant 与 AetherEdge 的本地同步，也不停止已经投运的
本地规则和安全行为。

## 未来反向暴露

把 AetherEdge 原生设备暴露给 Home Assistant 是另一条数据方向，必须作为独立的
Home Assistant 自定义集成交付，不能塞入
`extensions/home-assistant-bridge`。该自定义集成使用 AetherEdge 的公开应用
接口和独立凭据，不直接读取 SHM、配置文件或内部存储。

反向集成必须默认排除来源为 Home Assistant 的委托实体，避免把实体重新导出到
其来源。跨边界事件和命令还必须携带稳定事件标识、来源、路径和跳数上限，并按
幂等键去重。收到自身来源、重复路径或超过跳数上限的消息时必须拒绝，从契约层
阻断以下循环：

```text
Home Assistant -> AetherEdge -> Home Assistant -> AetherEdge
```

反向暴露拥有独立的权限、风险评审、版本和发布节奏。本决策不授权实现或启用它。

## 完整集成的验收门槛

只有同时满足以下条件，文档和发行说明才能把该能力标记为完整集成：

1. 区域、设备、实体、状态、可用性和能力目录能够完整初始化，并正确处理新增、
   重命名、移动、删除和重新加载；
2. 每个实体都有稳定身份和受限异构表示，不支持写入的实体仍可安全只读；
3. 常用可控实体的映射配置通过统一一致性测试，未知动作默认拒绝；
4. 重启、断网、认证撤销、权限变化、队列过载和接口错误不会造成静默漂移；
5. 命令测试覆盖拒绝、超时、重复、延迟状态、服务错误、确认未知和安全重试；
6. 本地密钥不会出现在云端、日志、审计明文或智能体上下文；
7. 数值投影与完整异构状态的权威边界可以由测试证明；
8. 公共跨产品数据在 AetherContracts 中先有有效样例、无效样例和一致性测试，
   Edge 与 Cloud 再消费同一固定版本；
9. 默认测试不需要外部服务，另有显式启用的真实 Home Assistant 端到端测试覆盖
   重连、重同步、删除墓碑和命令证据；
10. 能力状态页明确区分已实现、实验性和规划中内容。

## 实施状态与顺序

在本决策接受时，上述桥接扩展和厂商中立核心契约尚未作为已发布能力交付。实施
顺序为：

1. 先在 AetherContracts 定义跨产品可见的拓扑、异构值、能力和命令证据，并以
   样例和一致性测试冻结行为；
2. 在 AetherEdge 以行为测试驱动新增厂商中立领域类型、端口和应用用例；
3. 实现 `extensions/home-assistant-bridge`，通过模拟接口和扩展一致性测试；
4. 由 `services/io` 增加默认关闭的组合入口、密钥引用和运行诊断；
5. 增加显式启用的真实 Home Assistant 端到端验证；
6. 公共契约稳定后再增加 CloudLink 上报和 AetherCloud 派生目录；
7. 如有需求，再单独决策和实现反向自定义集成。

每一阶段都必须如实标记状态。完成某个基础类型或连接实验不等于完成产品集成。

## 后果

正面后果：

- AetherEdge 可以复用 Home Assistant 的广泛设备覆盖，同时保持行业中立核心；
- 智能体看到统一的设备、状态和能力契约，不需要学习厂商服务调用细节；
- 所有物理写操作继续经过同一安全和审计边界；
- 完整异构状态与高性能数值数据平面各自保留清晰职责；
- 本地同步不依赖 AetherCloud，云端故障不会破坏现场行为。

代价和限制：

- 经 Home Assistant 委托接入的设备在该路径上依赖其本地实例可用性；
- 完整目录、异构值和命令证据需要新增核心契约及跨仓库协调；
- 桥接需要维护接口版本兼容、重同步、背压和敏感属性策略；
- 能力映射必须持续扩展并通过一致性测试，不能以任意服务代理逃避维护成本；
- 同一设备若同时通过原生协议和 Home Assistant 接入，必须在投运时选择主来源，
  系统不会自动合并两个权威。

## 备选方案

- **让 AetherCloud 直接连接 Home Assistant**：否决。它会把现场凭据和实时控制
  依赖带入云端，破坏离线行为和 Edge 最终策略决定权。
- **把 Home Assistant 当作普通数值协议通道**：否决。它无法表达区域、设备、
  异构状态、能力和动态目录，也会制造错误的 SHM 权威。
- **把适配代码写进核心或 `services/io`**：否决。它会让厂商数据类型和客户端
  依赖污染核心；`services/io` 只负责选择并装配扩展。
- **公开任意 `call_service`**：否决。它绕过能力治理，无法可靠声明风险、
  幂等、前置条件和完成证据。
- **在同一个桥接扩展中同时做双向同步**：否决。两个方向的身份、权限、发布周期
  和循环风险不同，必须分离。
