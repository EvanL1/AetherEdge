# 典型的用户旅程

选择与您的角色匹配的最短路径。默认情况下，所有写入路径均保持受管制和拒绝状态。

## 评估本地边缘运行时

1. 打开[AetherEdge 概述](/aetheredge)。
2. 启动不带设备或外部服务的安全空组合。
3. 检查运行时运行状况和机器可读清单。
4. 仅在以下情况下添加协议适配器和域包：应用程序需要它。

## 构建边缘应用程序

1. 从正在运行的 AetherEdge OpenAPI 合约生成客户端。
2. 启动只读并保留质量、新鲜度、拓扑生成和修订字段。
3. 使用经过身份验证的应用程序边界；切勿直接编写 SHM 或 SQLite。
4. 仅添加具有显式权限、确认、幂等性和审核行为的受控命令。

## 将边缘队列连接到云

1. 在[兼容性矩阵](/compatibility/version-matrix) 中选择经过测试的组合。
2. 验证两者中摘要固定的 AetherContracts 消费者锁产品。
3. 遵循 [Edge to Contracts to Cloud 教程](/tutorials/edge-contracts-cloud)。
4. 保持 CloudLink 实验性和旧路径可用，直到每个已发布的发布通道都通过。

## 实现独立的客户端或运行时

1. 阅读 [AetherContracts概述](/aethercontracts)。
2. 实施规范规范和封闭的 Schemas。
3. 执行公共测试夹具和黑盒 TCK。
4. 报告一致性证据，无需声明产品部署或生产认证。

## 采用 AetherEMS

使用AetherEMS 当期望的结果是能源管理解决方案而不是通用边缘平台时。 AetherEMS 提供能源语义和工作流程，同时平台产品保持行业中立的边界。
