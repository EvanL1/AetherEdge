# Aether HTTP 历史查询

此可选扩展将语义数据处理历史功能映射到包含已具体化节奏网格的仅环回批处理端点。它仅接受任务策略 `aggregation=last` 和 `duplicate_policy=reject`；它不会聚合原始观察结果。

映射的范围由确切的任务和绑定标识确定。一个物理上游系列可以服务于多个任务，而每条 `(task, binding, feature)` 路由都是唯一的。

默认的 `aether-history` 收集流是原始的，因此不是通过此适配器进行重新采样的预测任务的有效来源。嵌入式 SQLite 适配器执行该生产聚合。此 HTTP 适配器适用于其上游服务已实现精确网格并可维护相同合约的部署。

线路响应与事件时间对齐；它不包含摄取削减或源/配置纪元。因此，适配器不会声明时间点回测语义，除非上游独立冻结并证明 v1 合约之外的那些削减。

适配器拒绝重定向、凭据、查询字符串、片段、非环回明文端点、过大的响应、不完整的映射、重复的后端点以及请求的半开窗口之外的时间戳。日历功能是根据请求的网格确定性生成的；所有其他功能都需要明确的委托映射。
