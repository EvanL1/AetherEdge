# 资源模型

> 权威来源：[AetherCloud](https://github.com/EvanL1/AetherCloud/blob/main/docs/concepts/resource-model.md)。此页面镜像到统一的 AetherIoT 文档中。

AetherCloud 使用显式 IoT 名词而不是通用实体表。该模型可以稍后获得类型化关系，但基本所有权和身份层次结构仍然可见。
```text
Tenant
└── Project
    ├── CloudConnection
    │   └── DeploymentStack
    │       └── CloudResourceProjection
    └── Site
        └── Gateway
            └── Instance
                └── Point
```

## 核心资源

**租户**是安全、计费、保留和数据隔离边界。每个拥有的聚合和事件都包含一个租户身份。

**项目**对应用程序或运营团队的站点和工件进行分组。它不能替代租户授权。

**CloudConnection** 引用一个明确选择的提供商身份和一个短期凭证源。它记录配置和发现状态，而不是原始的长期凭据。

**DeploymentStack** 是最小的独立规划、应用、锁定和恢复的基础设施单元。它只属于一个云连接，并且只使用一个远程状态绑定。它的租户、项目、提供者、连接和状态键是派生的，而不是作为调用者提供的值重复。区域放置保留其观察时间以用于未来的新鲜度政策。域定义已实现；持久性和生命周期执行已计划。

**CloudResourceProjection** 是 AetherCloud 提供程序资源的标准化、带时间戳的视图。基础设施提供商对于资源的实际存在和生命周期状态仍然具有权威。

**站点**是一个物理或逻辑操作位置，在网关被替换时仍然有意义。

**网关**是一个注册的 AetherEdge 运行时身份。重新安装或硬件更换必须遵循明确的身份恢复工作流程。

实施的网关基础区分注册、短期注册声明和声明消耗。活动凭证绑定、CloudLink 会话、运行时运行状况、撤销和恢复仍然是单独的计划事实。

**实例**是由网关报告的边缘事物模型实例。其云身份包括其网关上下文；边缘局部数字标识符不是全局唯一的。

**Point** 是实例内的类型化测量、状态、命令或调整点。云存储元数据和历史预测，而不是权威的实时值。

## 身份规则

- 公共资源身份是不透明的字符串，通常是 UUID。
- 边缘本地序列值和标识符在线路契约中保留其原始整数宽度。
- 显示名称是可变的，并且从不用作持久外键。
- 提供者身份是显式的，稳定的小写标识符；它们永远不会从凭证形状中推断出来。
- 可移植功能使用核心名称。特定于提供商的功能使用命名空间标识符，因此通用模型保持开放式。
- 在租户之间移动网关是一个明确的安全敏感工作流程，而不是普通更新。
- 域包添加类型化元数据和知识；它们不会取代核心所有权层次结构。

## 状态词汇

将 `desired` 用于云创作的目标，`reported` 用于边缘观察，仅当边缘报告成功激活时才使用 `applied`。使用 `connected` 作为最近经过身份验证的会话，而不是作为网关后面的设备正常运行的证据。

添加新的州名称之前，请参阅[术语参考](/aethercloud/reference/terminology)。请参阅[网关身份和注册](/aethercloud/concepts/gateway-identity-and-enrollment) 了解已实施的基础和明确的排除。
