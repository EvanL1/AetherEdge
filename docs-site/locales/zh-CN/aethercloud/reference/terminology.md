# 术语

> 权威来源：[AetherCloud](https://github.com/EvanL1/AetherCloud/blob/main/docs/reference/terminology.md)。此页面镜像到统一的 AetherIoT 文档中。

**应用的修订版** — 边缘报告已成功激活的修订版。它不是从下载或云请求中推断出来的。

**功能** - 通过应用程序层公开的类型化查询或命令，带有权限和安全元数据。

**功能作业** - 边缘功能的持久云意图，包括过期、幂等性、先决条件、授权证据和审核上下文。

**云连接** - 选择一个显式提供商身份、帐户的租户范围配置或订阅范围，以及秘密引用。

**CloudLink** — AetherCloud 和 AetherEdge 运行时之间的版本化、经过身份验证的协议。它不是一种设备协议。

**控制平面单元** — 一个独立运行的 API、CloudLink、工作线程、PostgreSQL 和对象存储放置，具有一个权威的事务写入器拓扑。租户拥有明确的归属小区。

**已连接** — 网关拥有足够新的经过身份验证的 CloudLink 会话。这并不意味着下游设备处于在线状态。

**所需修订版** — 网关或部署组的云创作目标修订版。

**部署堆栈** — 独立规划、锁定、应用、恢复和销毁的最小提供商范围单元，具有一个远程状态。其隔离密钥源自其租户、项目、云连接和堆栈身份。

**Edge** — 一个委托的 AetherEdge 运行时及其本地权限。边缘和网关在协议合约中相关，但不可互换。

**注册声明** - 租户/项目/网关绑定的、过期的引导程序声明，用于将注册的网关绑定到凭据请求证据。它不是长期有效的 API 或 CloudLink 凭证。

**网关凭证生成** — 一代独立可撤销的长期有效网关身份材料。恢复创建新一代。

**网关** — 代表注册的 AetherEdge 运行时身份的云资源。

**实例** — 一个网关拥有的事物模型实例。

**基础设施引擎** — 规划版本化提供程序模块的适配器。 OpenTofu 是默认值； Terraform 通过同一应用程序端口兼容。实现的端口故意不公开任何 Apply 操作。

**保存的计划** — 引擎生成的、针对一个部署堆栈和一个状态的不可变提案。 AetherCloud 保留加密的工件引用、摘要、锁定证据、版本、更改摘要和策略结果；批准或应用是一个单独的事实。

**计划工件** — 包含引擎已保存计划的加密外部对象。其二进制和原始 JSON 可以包含敏感值，并且永远不属于日志、提示、审计负载或规范化记录。

**放置** — AetherCloud 所需的提供商连接、区域和工作负载限制选择。它不能证明资源存在。

**Point** — 实例的类型化测量、状态、命令或调整成员。

**Projection** — 从权威边缘或提供商事实派生的 AetherCloud 表示形式，例如遥测历史记录、应用状态或云库存。

**Provider Adapter** — 功能驱动的插件，拥有提供商身份验证、发现、提供商特定模块、标准化观察结果和一致性证据。

**提供商数据库配置文件** — 托管或自托管 PostgreSQL 展示位置的提供商特定证据，包括版本、HA、备份、加密、网络、扩展、副本、区域、RPO/RTO、成本和本机扩展。它不是应用程序持久性合约。

**报告的状态** - 带时间戳的边缘观察。它可能会变得过时，并且不会覆盖所需的状态。

**站点** — 可以包含网关的持久物理或逻辑位置。

**状态** — 基础设施引擎在部署堆栈和提供程序资源之间的持久映射。它是远程的、锁定的、版本控制的、加密的，并且从不在提供商连接之间共享。

**租户** — 主要安全和数据隔离边界。

使用[资源模型](/aethercloud/concepts/resource-model) 来制定所有权和身份规则。
