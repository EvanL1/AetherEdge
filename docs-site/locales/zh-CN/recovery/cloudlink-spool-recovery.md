# 云端连接待发送队列恢复

当文件型待发送队列无法打开、检测到损坏、锁无法释放或重连后无法推进时，使用本手册。

该队列保存数据流代次、位置、稳定批次身份与摘要、发布证据、持久应用确认以及有界丢数证据。只有消息代理发布确认时，绝不能删除记录。

## 自动恢复边界

进程崩溃后，文件适配器只能截断未完整写入的日志尾部，然后恢复到最后一条完整并同步的记录。完全相同的应用确认可以幂等重放。

校验失败、语义冲突、不安全文件权限、符号链接、多重硬链接或完整记录损坏都必须安全失败，适配器不会猜测或静默跳过。

## 运维恢复

1. 停止所有可能持有队列的进程。不得强行破坏正在使用的独占锁，也不得让两个发布者共享同一路径。
2. 原样保存队列、锁、游标、挑战记录、配置和相关日志，并计算摘要。
3. 核对父目录仅对运行身份开放，文件所有者正确，且路径没有符号链接或多重硬链接。
4. 如果只是尾部写入不完整，可启动同一个适配器一次，并确认它恢复到最后一条完整记录。
5. 如果是校验或语义损坏，继续关闭发布。只有备份的数据流代次和位置历史属于同一网关与数据流时，才能恢复该备份。
6. 记录确实无法恢复时，只能使用协议规定并经过审查的丢数证据流程。不得手工修改位置、摘要或确认状态。
7. 重连后核对每条重放记录的原始位置、批次身份、业务摘要、发送时间和过期时间。只有收到精确匹配的持久应用确认后才能删除记录。

没有已验证备份或协议允许的丢数证据路径时，应停止并交由人工处理。丢失遥测数据也比伪造连续持久确认更安全。
