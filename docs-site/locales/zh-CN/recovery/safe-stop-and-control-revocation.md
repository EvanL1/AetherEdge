# 安全停止与控制撤销

当智能体、云端会话、集成、规则或通道不应继续请求物理动作时，使用本手册。

AetherEdge 可以停用具体资源，但尚未提供一个覆盖所有设备的通用软件急停能力。现场已经投运的物理急停、设备联锁和运行规程仍是紧急危险处置的最高权威。

## 隔离步骤

1. 人员或设备可能处于危险中时，先执行现场安全规程或物理联锁。
2. 在权限所属的权威边界撤销调用者权限或凭据。
3. 关闭范围最小的受影响控制面：家庭自动化控制功能、动作路由、规则或通信通道。
4. 只能使用带当前修订号、明确确认和持久审计的类型化应用命令。不得直接写共享内存、数据库、提供者接口或设备寄存器。
5. 保存操作身份、请求编号、结果修订号、终态审计和运行时对账状态。

命令超时、响应丢失、审计不完整或运行时收敛未知时，不得自动再次提交。应读取权威期望状态、资源状态、审计记录和物理观测，并保持控制路径关闭或物理隔离，直到人工消除证据冲突。

提供者接受、消息代理确认和边缘命令回执都不能证明物理过程已经进入安全状态。必须按照站点投运规则检查相关传感器或设备反馈。
